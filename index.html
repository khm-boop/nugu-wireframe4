  • 배송정보: 실제 어드민처럼 배송묶음코드 + 배송사 + 송장번호 + [배송추적][완료✓][삭제×]. 송장번호 입력은 테이블이 아니라 하단 [송장입력] 버튼에서 처리(테이블에서 입력칸 제거).
  • 상품정보 / 상품상태 / 할인내역: 썸네일·영문명·코드·일본어 링크·품목코드·옵션·취소선 원가·최종가·교환생성렉(←원주문)까지 실제 어드민 정보 누락 없이 반영. 상태 배지는 배송완료/배송중/취소완료/교환완료/교환철회 등 케이스별.
  • 전체(주문 단위) + SG 단위 2단 처리: SG 카드 하단에서 SG별 처리(부분취소/반품/교환/송장입력), 상단 전체 바에서 주문 단위 최종 결제취소/환불 확정. 회의록의 "결제는 한번에 취소" 반영. 페이페이 부분환불은 검토 필요.
  • 결제 요약 세분화: 상품금액 → 쿠폰 할인액(SG 안분 펼침) → 배송비(유형별 분해) → 수수료 → 합계 → 결제 → 환불(SG별) → 포인트 → 쿠폰코드. 미확정 항목은 검토 필요.
